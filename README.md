# Claude-SKILL

รวม **Claude Code Skills** สำหรับงานเอกสารและงานสำนักงานของหน่วยงานภาครัฐไทย (พัฒนาและใช้งานจริงที่ทีม DGA-SD)

## จุดประสงค์

- รวบรวม skill ที่ใช้ซ้ำได้กับ [Claude Code](https://claude.com/claude-code) ให้ติดตั้งง่าย ใช้ได้ทุกเครื่อง (macOS / Linux / Windows-WSL)
- เน้นงานที่ทีมทำบ่อย เช่น สรุปรายงานการประชุมจาก transcript อัตโนมัติของ MS Teams ให้เป็นรายงานภาษาไทยพร้อมไฟล์ Word
- ออกแบบให้ **ไม่มีข้อมูลส่วนบุคคลใน repo** — ไฟล์รายชื่อบุคคล (roster) เป็น optional และเก็บเฉพาะในเครื่องผู้ใช้เท่านั้น (ดูหัวข้อ PDPA ด้านล่าง)

## ที่มา

skill ชุดนี้เริ่มจากงานสรุปรายงานการประชุมของ DGA ที่ต้องทำประจำ: transcript ที่ได้จาก MS Teams เป็นการถอดเสียงอัตโนมัติซึ่ง**ชื่อคนและศัพท์เทคนิคเพี้ยนมาก** และการจัดรูปแบบรายงาน (ฟอนต์ราชการ TH SarabunPSK, โครงสร้างวาระ, มติ, action items) กินเวลา จึงทำเป็น skill ให้ Claude Code ทำงานนี้ให้จบในคำสั่งเดียว แล้วขยายเป็น repo กลางสำหรับ skill อื่น ๆ ที่จะตามมา

## การติดตั้ง (ภาพรวม — ใช้กับทุก skill)

1. Clone repo นี้:
   ```bash
   git clone https://github.com/DGA-SD/Claude-SKILL.git
   ```
2. Copy โฟลเดอร์ของ skill ที่ต้องการไปไว้ที่โฟลเดอร์ skills ของ Claude Code ในเครื่องคุณ:
   ```bash
   mkdir -p ~/.claude/skills
   cp -r Claude-SKILL/meeting-summary ~/.claude/skills/
   ```
3. เปิด Claude Code แล้วพิมพ์ `/` — ชื่อ skill จะปรากฏในรายการ

> ติดตั้งเฉพาะบางโปรเจกต์ก็ได้ โดย copy ไปไว้ที่ `<โปรเจกต์>/.claude/skills/` แทน

### 1. meeting-summary

**คืออะไร:** แปลง transcript การประชุมดิบ (ถอดเสียงอัตโนมัติจาก MS Teams) เป็น**รายงานสรุปการประชุมภาษาไทย** โครงสร้างครบ: หัวเรื่อง / วัน-เวลา / ผู้พูดหลักและบทบาท / ที่มา / สิ่งที่นำเสนอรายวาระ / ช่วงถาม-ตอบ / Action Items / เรื่องอื่น ๆ — และสร้างไฟล์ให้ **2 รูปแบบพร้อมกัน** คือ `.md` และ `.docx` (ฟอนต์ TH SarabunPSK ตัวเนื้อหา 16pt หัวข้อไล่ขนาดอัตโนมัติ ตัวเลขอารบิกทั้งฉบับ) จุดเด่นคือช่วย**แก้ชื่อคน/ศัพท์เทคนิคที่ถอดเสียงเพี้ยน**ให้ถูกต้องตามบริบท และ flag ชื่อที่ไม่แน่ใจให้ผู้ใช้ยืนยันท้ายรายงาน

**ข้อกำหนดของเครื่อง:**
- `python3` + แพ็กเกจ `python-docx` (`pip3 install python-docx`)
- ฟอนต์ **TH SarabunPSK** ติดตั้งในเครื่องที่จะ*เปิดอ่าน*ไฟล์ .docx

**ติดตั้ง:**
```bash
cp -r Claude-SKILL/meeting-summary ~/.claude/skills/
pip3 install python-docx
```

**วิธีใช้:**
1. เตรียมไฟล์ transcript เป็น `.txt` หรือ `.md` (ถ้ามีแต่ `.vtt` ให้เปิดไฟล์ transcript `.docx` ที่ดาวน์โหลดจาก MS Teams → copy ข้อความทั้งหมด → วางลงไฟล์ `.txt` — skill ไม่รับ `.vtt` โดยตรงเพราะเปลือง token มาก)
2. ใน Claude Code สั่ง:
   ```
   /meeting-summary สรุปรายงานการประชุมจากไฟล์ ./meeting/transcript-ประชุม-X.txt
   ```
3. ตรวจรายชื่อผู้พูดที่ skill flag ว่าไม่แน่ใจ แล้วยืนยัน/แก้ไขกลับไป

**ตัวอย่างการใช้:**
```
/meeting-summary รบกวนสรุปรายงานการประชุม SC 4-2569 จากไฟล์ transcript นี้ ./meeing/SC-4-2569-meeting-transcript.txt
```
ผลที่ได้: รายงานสรุปครบทุกวาระ พร้อมตารางผู้พูดหลัก ตาราง Action Items และรายการชื่อที่ขอให้ผู้ใช้ช่วยยืนยันท้ายบทสนทนา

**ไฟล์ผลลัพธ์อยู่ที่ไหน:** สร้างใน**โฟลเดอร์เดียวกับไฟล์ transcript ต้นทาง** เสมอ (ไม่ส่งข้อมูลออกนอกเครื่อง):
```
สรุปการประชุม-<หัวข้อ>.md
สรุปการประชุม-<หัวข้อ>.docx
```

**Optional — ไฟล์รายชื่อบุคคล (roster) เพื่อให้แก้ชื่อแม่นขึ้น:**
transcript อัตโนมัติมักถอดชื่อคนเพี้ยน skill จะแก้ชื่อได้แม่นยำขึ้นมากถ้าคุณสร้างไฟล์รายชื่อขององค์กรไว้ที่:
```
~/.claude/skills/meeting-summary/references/dga-people.md
```
วิธีสร้าง: copy เทมเพลตจาก `references/dga-people.example.md` (ใน repo นี้ — ชื่อในไฟล์เป็นชื่อสมมติทั้งหมด) แล้วแทนที่ด้วยรายชื่อจริงขององค์กรคุณ ภายหลังมีคนเพิ่ม/ชื่อผิด ก็แก้ที่ไฟล์นี้ไฟล์เดียว
ถ้า**ไม่มี**ไฟล์นี้ skill ยังทำงานได้ตามปกติ โดยอาศัยป้ายชื่อผู้พูดภาษาอังกฤษใน transcript + บริบทการประชุม แล้ว flag ชื่อที่ไม่แน่ใจให้ผู้ใช้ยืนยันแทน

## หมายเหตุด้าน PDPA / ความปลอดภัยข้อมูล

- **ห้าม commit ไฟล์ `references/dga-people.md` (รายชื่อบุคคลจริง) ขึ้น repo** — เป็นข้อมูลส่วนบุคคลตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (PDPA) ไฟล์นี้ต้องอยู่เฉพาะในเครื่องผู้ใช้ (`.gitignore` ของ repo นี้กันไว้แล้ว แต่โปรดระวังเมื่อนำไปใช้ที่อื่น)
- transcript การประชุมมีข้อมูลส่วนบุคคล — skill ถูกออกแบบให้ทำงาน**ในเครื่องเท่านั้น**: ไม่อัปโหลด transcript หรือผลสรุปไปบริการภายนอก และไฟล์ผลลัพธ์อยู่ในโฟลเดอร์เดียวกับ transcript
- skill ปฏิบัติต่อเนื้อหา transcript เป็น "ข้อมูล" เท่านั้น ไม่ใช่ "คำสั่ง" เพื่อป้องกัน prompt injection (OWASP LLM01)
