🚗 mDL — แสดงใบขับขี่ดิจิทัลแบบ Offline

ISO/IEC 18013-5 · mdoc · Device Engagement (QR) + BLE · ติดตามการเคลื่อนที่ของข้อมูลตามเส้นทาง
🌐 เฟสเตรียมล่วงหน้า · ออก mDL + โหลด Trust List (ต้องมีเน็ตเป็นครั้งคราว)
✈️ เฟส ณ จุดตรวจ · Offline 100% — QR + BLE เท่านั้น

เฟสเตรียมล่วงหน้า (ต้องออนไลน์เป็นครั้งคราว): ก่อนจะตรวจแบบ offline ได้ ทั้งสองฝั่งต้อง "ตุนเสบียง" ไว้ก่อน — ฝั่ง Holder ขอออก mDL เก็บเข้ากระเป๋า (กุญแจอุปกรณ์เกิดในชิปนิรภัยของมือถือ และไม่ออกจากเครื่อง) ส่วนฝั่งเครื่องตรวจโหลด Trust List (ใบรับรองราก IACA) เก็บไว้ในเครื่อง — หลังจบเฟสนี้ การพบกัน ณ จุดตรวจไม่ต้องใช้อินเทอร์เน็ตอีกเลย.

ลำดับการทำงาน

    เฟสนำเสนอ ณ จุดตรวจ (offline 100%): ข้อมูลเดินทางผ่าน QR (แสง) และ BLE (คลื่นวิทยุระยะใกล้) เท่านั้น — ไม่มีฝั่งใดเชื่อมอินเทอร์เน็ต ความน่าเชื่อถือมาจากลายเซ็นที่พกมาในตัว mDL (MSO ของ Issuer) บวกกับ Trust List ที่โหลดไว้ล่วงหน้า — โซนออนไลน์ด้านบนถูกหรี่ไว้ เพราะไม่ถูกแตะเลยในเฟสนี้.

    ✈️ OFFLINE — ไม่แตะอินเทอร์เน็ต

    ลำดับการทำงาน

      Device Key (เกิดในชิปนิรภัย) คำขอ (Request) mDL / mdoc (ข้อมูล + ลายเซ็น Issuer) ใบรับรอง IACA / Trust List QR — Device Engagement BLE Secure Session ยินยอม + Selective Disclosure ผลการตรวจสอบ (ในเครื่อง)

      อ้างอิง ISO/IEC 18013-5 (mobile Driving Licence — mdoc) ซึ่งมาตรฐาน TGIX VC/VP รองรับควบคู่กับ SD-JWT. ศัพท์สำคัญ: MSO (Mobile Security Object — ชุด digest รายฟิลด์ + device public key ที่ Issuer เซ็นกำกับ) · IACA (Issuing Authority CA — ใบรับรองรากของผู้ออก) · VICAL (บัญชีรวมใบรับรอง IACA จากหลายผู้ออก) · DeviceAuth (ลายเซ็นจาก device key ในชิปนิรภัย = holder binding). ข้อสังเกตของโหมด offline: เช็ค "สถานะสด" (เพิกถอน ณ วินาทีนี้) ไม่ได้ — ระบบจึงคุมความสดด้วยอายุของ MSO (Issuer เซ็นออกใหม่เป็นระยะ เมื่อหมดอายุเครื่องตรวจจะไม่ยอมรับ).
